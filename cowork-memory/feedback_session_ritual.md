Do NOT create a new *-HANDOVER-*-EOD.md file at the start of a new session just to record context.

**Why:** Paddy said it explicitly — "sack putting a new file at the start of every chat. its killing me."

**How to apply:**
- At session start: read the latest EOD handover from GitHub (asgard-handovers repo or project repo docs/HANDOVER.md) to get context, then get on with it. Don't write anything.
- At session END (when Paddy says "end session", "wrap up", or "save state"): write/overwrite a single canonical file (no date prefix), post summary to Discord, then done.
